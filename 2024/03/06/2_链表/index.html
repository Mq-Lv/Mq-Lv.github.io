2_链表

总结：

  • 链表类题目一般都有两种方法：
    • 直接使用原来的链表来进行操作
    • 设置一个虚拟头节点再进行操作
  • 注意双链表的使用
  • 一般将虚拟头节点作为cur，之后虚拟头节点不变，从而保证虚拟头节点.next即为要返回的head

1、基础理论

  • 链表：通过指针串联起来的线性结构，每一个节点由两部分组成：数据+指针

    链表的入口节点称为链表的头节点（head）

1.1、链表的类型

  • 单链表

    指针指向下一个节点，最后一个节点的指针为null

  • 双链表：每一个节点有两个指针域，一个指向上一个节点，一个指向下一个节点。即可以向前查询又可以向后查询

  • 循环链表：可以解决约瑟夫环问题

1.2、链表的存储方式

链表通过指针域的指针连接内存中的各个节点，所以在内存中不是连续分布的，而是散乱的分别在内存中的某地址上，分配机制取决于操作系统的内存管理

2、经典题目

2.1、移除链表元素

思路：（两种）

  • 直接使用原来的链表来进行删除操作
    • 先对头节点进行判断和移动
  • 设置一个虚拟头节点再进行删除操作
    • 设置一个虚拟的头节点指向当前头节点，此时头节点就可以与其余节点一样操作
    • 随后分别设置一个prevNode和curNode。通过循环判断curNode来完成移除操作

2.2、设计链表

在链表类中实现这些功能：

  • get(index)：获取链表中第 index 个节点的值。如果索引无效，则返回-1。
  • addAtHead(val)：在链表的第一个元素之前添加一个值为 val 的节点。插入后，新节点将成为链表的第一个节点。
  • addAtTail(val)：将值为 val 的节点追加到链表的最后一个元素。
  • addAtIndex(index,val)：在链表中的第 index 个节点之前添加值为 val 的节点。如果 index 等于链表的长度，则该节点将附加到链表的末尾。如果 index 大于链表长度，则不会插入节点。如果index小于0，则在头部插入节点。
  • deleteAtIndex(index)：如果索引 index 有效，则删除链表中的第 index 个节点。

2.4、两两交换链表中的节点

思路：

  1. 引入一个虚节点prev，定义节点cur = head
  2. 判断cur和cur.next是否都为非空
  3. 如果都非空，则存储temp = cur.next.next，令nex = cur.next指向cur，==cur指向null==，prev指向nex。
  4. prev = cur, cur = temp，
  5. 如果仅cur非空，则令prev指向cur

总结：

  • 注意虚节点的使用
  • 注意当交换链表中的方向改变时需要令之前的指向为null，来防止在链表中出现环

2.5、删除链表的倒数第N个节点

思路：快慢指针、移动窗口

  • 双指针的经典应用，如果要删除倒数第n个节点，让fast移动n步，然后让fast和slow同时移动，直到fast指向链表末尾。删掉slow所指向的节点就可以了。
  1. 使用虚拟头结点
  2. 定义fast指针和slow指针，初始值为虚拟头结点
  3. fast首先走n + 1步 ，为什么是n+1呢，因为只有这样同时移动的时候slow才能指向删除节点的上一个节点（方便做删除操作）
  4. fast和slow同时移动，直到fast指向末尾
  5. 删除slow指向的下一个节点
